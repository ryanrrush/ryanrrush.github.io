Accessibility Checklist

Website Page Titles

When creating webpages, there is a <title> code within web pages that states what the title of the page is. This is often presented at the top of the browser tab. If the mouse is rolled over this, the full page title is often displayed. It is important to have a good title structure in your webpages so that a perosn navigating your site will properly know which page they are on.

Results: 14 pages on the John Spoor Broome Library website were evaluated for clear, unique HTML page titles. Each page has descriptive HTML title values. We do recommend that all the title values be formatted in a similar fashion. The home page has an excellent title structure as "Home -John Spoor Broome Library - CSU Channel Islands " whereas others like the course reserves page just says “Course Reserves”. We recommend all page titles follow a structure like the home page.

Image Text Alternatives

Items like pictures, illustrations, charts, etc. require alternative text. This is text that describes what the important information this particular object is supposed to convey. For individuals using a screen reader, they may not be able to visually see this item and rely on the alternative text to read back to them what information is contained within the image, illustration, chart, etc.

Results: There are some photos on the home page that have no alternative text. These are for the Study Rooms, Ask a Librarian, and Equipment Checkout sections. This may make it difficult for patrons using an assistive technology to know what the photo is. If these do not provide meaningful information, the term "decorative" may be appropriate. Photos on other pages have alternative text included.

Text:

The headings, color contrast, and text resizing are important aspects of accessibility for web pages. These assist users who might otherwise have difficulty seeing or interacting with the text.

Results - Headings On the home page there is no H1 tag. The rest of the pages searched all had proper headings starting with H1. A possible spot for the H1 tag on the home page could be "John Spoor Broom Library" at the top of the page. This will allow proper navigation back to the top of the page for screen readers.

Results - Contrast Ratio Throughout the site we are getting warnings of color contrast errors. A visual inspection says this is incorrect. Upon a deeper inspection, it appears the error may come from a hidden color or the opacity settings on the red background and whtie text. This could be corrected within the HTML coding of the sections.

Results - Resizing Text Resizing text is important for individuals who have a difficulty seeing text on a webpage. The text on the various pages tested for the John Spoor Broome Library resulted in positive resizing results.

Interaction:

Keyboard access and visual focus

There are some who cannot use a mouse to navigate the webpage and need to rely on a keyboard for navigation. Keyboard navigation needs to be clear and follow a structured path through the elements.

Results: Using only keyboard keys to navigate the John Spoor Broome Library website yeilded excelent results. It was easy and simple to navigate various webpages using the tab key.

Forms, labels, and errors (including Search fields):

Forms include items like e-mail address boxes, login boxes, and search boxes. Each form needs to be labeled properly with a a label element as well as value and id attribute fields. For example:
  • Each form control has a label element with a for attribute that matches the value of the id attribute in the related control. For example:
    <label for="firstname">First name: </label>
    <input type="text" name="firstname" id="firstname" />
  • Each id is unique within the web page.

Results of the various pages that we tested, we only found the "search library" field at the top. This is keyboard accessible and has proper labels for assistive devices.

General:

Moving, Flashing, or Blinking Content can be difficult for individuals with certain visual disabilites. Things like videos, scrolling feeds, and even stock tickers can be difficult to navigate. Users need to be able to control this content using a keyboard.

Results: Of the various pages that we tested, we found no flashing, moving, or blinking content. If you plan to implement this type of content into your webpages, be sure to follow accessiblity guidelines similar to the ones at Easy Checks – A Frist Review of Web Accessibility for any flashing, moving, or blinking content.

Multimedia (video, audio) closed captions and transcripts

Results: We found no videos on the pages checked. If you plan to include videos on your webpages, be sure to follow accessiblity guidelines similar to the ones at Easy Checks – A Frist Review of Web Accessibility for videos. Videos will want to have proper and accurate closed captioning and transcripts.

Structural checks of a webpage allows individuals to see how content is ordered and accessed. By removing viewing a page without various layouts, styles, or even images is a way to check the flow of the webpage.

Results: Using the WAVE browser extension form Google Chrome, we were able to remove the CSS on your webpages. This allowed us a look at the structure of the page in a line and see how the headings were nested within one another, alternative text for photos, etc. If you would like more information about basic structural checks for your webpages, you can view them at Easy Checks – A Frist Review of Web Accessibility.
